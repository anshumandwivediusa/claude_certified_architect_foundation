# Multi-Agent Architecture: Coordinator / Sub-Agent Delegation Patterns

## Claude Architect Exam Notes

# Module Overview

As AI systems become more complex, a **single agent** may not be sufficient to solve large or specialized tasks efficiently. Instead, organizations use **Multi-Agent Architectures**, where multiple AI agents collaborate, each with a specific responsibility.

In Anthropic's Claude ecosystem, a common pattern is the **Coordinator–Sub-Agent Delegation Pattern**, where one agent orchestrates specialized agents and combines their results into a final response.

# What is a Multi-Agent Architecture?

## Definition

A **Multi-Agent Architecture** is a system in which multiple AI agents collaborate to accomplish a common goal. Each agent specializes in a specific domain or capability, while one or more agents coordinate the overall workflow.

Unlike a single agent that performs every task, a multi-agent system divides responsibilities among specialized agents.


# Why Multiple Agents?

A single agent may become overloaded with responsibilities such as:

* Searching documents
* Writing code
* Querying databases
* Planning tasks
* Reviewing results
* Sending notifications

Instead, these responsibilities can be distributed to specialized agents.

# Coordinator Agent

## Definition

The **Coordinator Agent** (also called the **Orchestrator** or **Supervisor**) manages the overall task.

It does **not necessarily perform every task itself**.

Instead, it:

* Understands the user goal.
* Breaks it into smaller tasks.
* Delegates tasks to sub-agents.
* Collects results.
* Resolves conflicts.
* Produces the final response.

Think of the coordinator as a **project manager**.


# Responsibilities of the Coordinator

* Goal decomposition
* Task planning
* Agent selection
* Delegation
* Monitoring progress
* Handling failures
* Combining results
* Returning the final answer

# MCP Integration

Each sub-agent may use different MCP servers.

```text
                   Coordinator

                         │

      ┌──────────────────┼──────────────────┐

      ▼                  ▼                  ▼

 Research Agent     Coding Agent     Database Agent

      │                  │                  │

 SharePoint MCP     GitHub MCP        SQL MCP

      │                  │                  │

      └──────────────────┼──────────────────┘

                         ▼

                  Final Response
```

The coordinator does not directly access every resource. It delegates to specialized agents that interact with the appropriate MCP servers.


# Advantages

* Separation of responsibilities
* Parallel execution
* Better scalability
* Easier maintenance
* Domain specialization
* Improved accuracy
* Reusable agents
* Independent development teams


# Challenges

* Increased orchestration complexity
* Communication overhead
* Higher infrastructure cost
* Result synchronization
* Error propagation
* Conflict resolution
* Security between agents
* Monitoring distributed execution


# Coordinator vs Sub-Agent

| Coordinator Agent        | Sub-Agent                                |
| ------------------------ | ---------------------------------------- |
| Understands overall goal | Performs specialized tasks               |
| Creates execution plan   | Executes assigned work                   |
| Delegates tasks          | Does not manage other agents (typically) |
| Tracks progress          | Reports results                          |
| Combines outputs         | Returns task-specific output             |
| Produces final response  | Focuses on one domain                    |


# When to Use Multi-Agent Architecture

Use a multi-agent system when:

* Tasks require different expertise.
* Independent work can be parallelized.
* Systems must scale across domains.
* Different enterprise systems are involved.
* Large workflows become difficult for a single agent.

Avoid it for simple tasks that a single agent can complete efficiently, as the added coordination may outweigh the benefits.


# Claude Architecture Perspective

In Claude-based systems:

* **Claude (Coordinator)** understands the user's goal, plans the work, delegates tasks, and synthesizes results.
* **Sub-agents** specialize in domains such as coding, research, security, or data analysis.
* **MCP servers** provide the external capabilities each sub-agent needs (Git, databases, Slack, filesystems, etc.).
* The coordinator manages the overall execution and returns a unified answer.


# Best Practices

* Keep coordinator logic focused on orchestration rather than domain work.
* Give each sub-agent a single, well-defined responsibility.
* Prefer parallel execution for independent tasks.
* Define clear input/output contracts between agents.
* Validate outputs before combining them.
* Implement retries, timeouts, and fallback strategies.
* Log delegation decisions for observability and auditing.


# Interview Questions

### Q1. What is a Multi-Agent Architecture?

**Answer:**
A Multi-Agent Architecture consists of multiple specialized AI agents that collaborate to achieve a common objective. A coordinator agent manages planning, delegation, and result aggregation.


### Q2. What is the role of the Coordinator Agent?

**Answer:**
The coordinator decomposes the user's goal into subtasks, assigns them to appropriate sub-agents, monitors execution, handles failures, and combines the results into a final response.


### Q3. What are sub-agents?

**Answer:**
Sub-agents are specialized AI agents responsible for specific domains or tasks, such as research, coding, database access, testing, or deployment.


### Q4. When is parallel delegation preferred?

**Answer:**
Parallel delegation is appropriate when subtasks are independent and can execute simultaneously, reducing overall execution time.


### Q5. How does MCP fit into a Multi-Agent Architecture?

**Answer:**
Each sub-agent can use one or more MCP servers to access external resources such as Git repositories, databases, Slack, or file systems. The coordinator delegates work, while the sub-agents perform tool interactions through MCP.
