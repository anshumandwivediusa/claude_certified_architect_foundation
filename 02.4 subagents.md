# How Sub-Agents Operate with Isolated Context

# Module Overview

In a **Multi-Agent Architecture**, the **Coordinator Agent** delegates work to specialized **Sub-Agents**.

A key design principle in Claude is:

> **Sub-agents do NOT automatically inherit the coordinator's conversation history or entire context.**

Instead, the coordinator provides **only the information required** for the assigned task.

This principle is known as **Context Isolation**.

# What is Context Isolation?

## Definition

**Context Isolation** means each sub-agent executes independently using only the context explicitly provided by the coordinator.

Sub-agents do **not** receive:

* Entire conversation history
* Previous reasoning steps
* Hidden chain-of-thought
* Memory belonging to other agents
* Internal planning unless deliberately shared

Instead, they receive a **minimal task package**.

# Challenges

Coordinator must

* Extract the correct context
* Avoid omitting essential information
* Balance minimal context with sufficient context
* Handle dependencies between agents

Poor context packaging can lead to incomplete or incorrect results.


# Coordinator vs Sub-Agent

| Coordinator               | Sub-Agent                           |
| ------------------------- | ----------------------------------- |
| Sees overall conversation | Sees only assigned task             |
| Maintains global context  | Maintains local context             |
| Plans execution           | Executes delegated task             |
| Delegates work            | Returns focused result              |
| Combines outputs          | Does not combine other agents' work |


# Best Practices

* Share only task-relevant information.
* Apply the Principle of Least Privilege.
* Define clear task inputs and outputs.
* Avoid exposing unrelated user data.
* Keep prompts concise and focused.
* Validate sub-agent results before merging.
* Log delegation decisions for auditing.

# Interview Questions

### Q1. Do Claude sub-agents inherit the coordinator's entire conversation history?

**Answer:**
No. Sub-agents operate with isolated context. They receive only the task-specific information explicitly provided by the coordinator.


### Q2. Why does Claude isolate sub-agent context?

**Answer:**
Context isolation improves efficiency, reduces token usage, enhances security and privacy, minimizes irrelevant information, and allows sub-agents to focus on their assigned responsibilities.


### Q3. What information is typically passed to a sub-agent?

**Answer:**
The coordinator provides the task description, relevant documents or data, constraints, expected output, and any necessary tool access. Unrelated conversation history is excluded.


### Q4. What is the coordinator's responsibility regarding context?

**Answer:**
The coordinator determines what information is relevant, packages it into a focused task, delegates it to the appropriate sub-agent, and later combines the sub-agent outputs into a final response.

### Claude Architect Key Takeaway

For the Claude Architect exam, **context isolation** is a deliberate architectural choice, not a limitation. It improves **security**, **privacy**, **performance**, **cost efficiency**, and **modularity** by ensuring that each sub-agent receives **only the minimal, task-specific context** required to complete its work, while the **coordinator** retains the complete conversation state and is responsible for orchestrating and synthesizing the overall result.
