# Escalation Triggers: When an AI Agent Should Hand Off to a Human

## Claude Architect Exam Notes

# Module Overview

No AI system should attempt to solve **every problem autonomously**.

In enterprise environments, Claude agents must recognize situations where **human intervention is required**. This process is called **escalation** or **human handoff**.

An escalation trigger is a condition that causes the coordinator to stop autonomous execution and transfer the task to a human, another business process, or a higher-authority system.

> **Key Principle:** A good agent knows **when not to act autonomously**.


# Learning Objectives

After this module, you should understand:

* What escalation means
* Common escalation triggers
* Difference between retry, replanning, and escalation
* Human-in-the-loop (HITL)
* Enterprise escalation workflows
* Best practices for safe autonomous systems


# What is Escalation?

## Definition

**Escalation** is the process of transferring responsibility from an AI agent to a human or another authorized process when the agent cannot safely, confidently, or appropriately continue.

The coordinator recognizes that autonomous execution has reached its limit.


# Why is Escalation Needed?

Enterprise AI systems must balance:

* Autonomy
* Safety
* Compliance
* Reliability
* Accountability

Instead of forcing an answer, the agent should escalate when appropriate.

# Retry vs Replanning vs Escalation

| Retry              | Replanning                  | Escalation                               |
| ------------------ | --------------------------- | ---------------------------------------- |
| Repeat same action | Choose a different strategy | Transfer to a human or another authority |
| Temporary failures | Alternative workflow        | AI cannot safely continue                |
| Example: timeout   | Example: use another MCP    | Example: manager approval                |

# Best Practices

* Respect explicit requests for a human.
* Define clear escalation criteria.
* Limit retries before escalating.
* Never invent missing policy.
* Log every escalation event.
* Preserve execution history for the reviewer.
* Clearly explain why escalation occurred.
* Resume automation only if authorized.


# Coordinator Responsibilities

The coordinator should:

1. Detect escalation triggers.
2. Stop autonomous execution safely.
3. Preserve task state and evidence.
4. Route the task to the correct human or business process.
5. Explain why escalation occurred.
6. Resume execution only when appropriate.

# Interview Questions

### Q1. What is escalation in a multi-agent system?

**Answer:**
Escalation is the controlled transfer of responsibility from an AI agent to a human or another authorized process when the agent cannot safely, confidently, or appropriately continue the task.


### Q2. What are common escalation triggers?

**Answer:**
Common triggers include:

* Explicit user request for a human
* Policy exceptions
* Policy gaps or ambiguity
* Inability to make progress
* Missing required information
* Low confidence
* Conflicting evidence
* Sensitive or high-impact decisions
* Security incidents
* Compliance requirements


### Q3. What is the difference between replanning and escalation?

**Answer:**
Replanning keeps execution within the AI system by changing the strategy or tools used. Escalation transfers responsibility outside the AI system to a human or another authority because autonomous execution is no longer appropriate.


### Q4. Why should an AI not invent policies?

**Answer:**
Inventing policies can lead to incorrect, unsafe, or non-compliant decisions. When policies are missing, unclear, or contradictory, the correct action is to escalate to the responsible human decision-maker.


### Q5. What information should accompany an escalation?

**Answer:**
The coordinator should provide:

* User request
* Work completed
* Current task state
* Evidence collected
* Errors encountered
* Reason for escalation
* Recommended next action

This allows the human reviewer to continue without restarting the investigation.

# Claude Architect Key Takeaway

For the Claude Architect exam, remember that **escalation is a feature, not a failure**. Mature AI systems recognize the limits of autonomous execution and safely hand off work when human judgment, organizational authority, or additional information is required.

A robust Claude-based system should:

* **Retry** transient failures.
* **Replan** when a different execution strategy is available.
* **Escalate** when it cannot safely or appropriately continue.

This distinction is central to designing trustworthy, enterprise-grade agentic systems.
