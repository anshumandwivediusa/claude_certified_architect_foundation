# Prompt Chaining vs. Dynamic Decomposition

## Matching the Right Pattern to the Right Task

### Claude Architect Exam Notes


# Module Overview

One of the most important architectural decisions in Claude-based AI systems is determining **how a complex task should be executed**.

There are two common orchestration patterns:

1. **Prompt Chaining (Fixed Sequential Steps)**
2. **Dynamic Decomposition (Open-Ended Investigation)**

Choosing the wrong pattern can result in:

* Higher costs
* Slower execution
* Reduced accuracy
* Unnecessary complexity

The Claude Architect exam often tests **when each pattern is appropriate**.


# Learning Objectives

After this module, you should be able to:

* Define Prompt Chaining.
* Define Dynamic Decomposition.
* Compare both patterns.
* Identify enterprise use cases.
* Select the appropriate architecture.
* Understand Claude's role in each pattern.


# What is Prompt Chaining?

## Definition

**Prompt Chaining** is an orchestration pattern where a complex task is divided into a **predefined sequence of prompts or steps**, with the output of one step becoming the input to the next.

The execution path is known **before execution begins**.


# Characteristics

* Fixed sequence
* Deterministic
* Predictable
* Easy to debug
* Same flow every time
* Minimal planning during execution


# Architecture

```text
          User Request
                │
                ▼
          Prompt 1
                │
                ▼
          Prompt 2
                │
                ▼
          Prompt 3
                │
                ▼
          Final Response
```


# Example

User

> Summarize this PDF and translate it into French.

Execution

```text
Read PDF

↓

Summarize

↓

Translate

↓

Return Result
```

The sequence is fixed.


# Enterprise Examples

* Document summarization
* Translation pipelines
* Invoice processing
* OCR → Extraction → Validation
* Resume parsing
* Content moderation
* Email generation


# Advantages

* Simple
* Predictable
* Easy to monitor
* Lower cost
* Easier testing
* Easier compliance


# Limitations

* Cannot adapt
* No autonomous planning
* Breaks when unexpected situations occur
* Limited flexibility


# What is Dynamic Decomposition?

## Definition

**Dynamic Decomposition** is an orchestration pattern in which the AI agent determines **at runtime** how to divide a complex goal into subtasks.

The execution plan is **not predefined**.

Claude continuously reasons about:

* What should happen next?
* Which tools are needed?
* Which sub-agent should execute?
* Is more investigation required?


# Characteristics

* Goal-driven
* Adaptive
* Runtime planning
* Autonomous
* Supports replanning
* Open-ended execution


# Architecture

```text
            User Goal
                 │
                 ▼
         Claude Coordinator
                 │
         Analyze Goal
                 │
     Create Task Plan Dynamically
                 │
        ┌────────┼────────┐
        ▼        ▼        ▼
   Research  Database  Coding
                 │
                 ▼
      Evaluate Results
                 │
          More Work?
            │     │
           Yes   No
            │     │
            ▼     ▼
     Create New Tasks
```


# Example

User

> Investigate why production performance degraded after yesterday's deployment.

Claude does not know the answer beforehand.

It may decide to:

```text
Review deployment logs

↓

Analyze metrics

↓

Check database

↓

Inspect Git commits

↓

Compare configurations

↓

Identify root cause
```

The plan emerges during execution.


# Enterprise Examples

* Root cause analysis
* Security investigations
* Software debugging
* Financial audits
* Incident response
* Competitive research
* Compliance investigations


# Advantages

* Flexible
* Adaptive
* Handles unknown problems
* Supports tool selection
* Supports iterative refinement
* Better for discovery tasks


# Limitations

* More expensive
* Less predictable
* Harder to test
* Requires stronger governance
* Longer execution


# Side-by-Side Comparison

| Feature        | Prompt Chaining  | Dynamic Decomposition |
| -------------- | ---------------- | --------------------- |
| Planning       | Before execution | During execution      |
| Execution Path | Fixed            | Dynamic               |
| Adaptability   | Low              | High                  |
| Tool Selection | Predetermined    | Runtime               |
| Replanning     | No               | Yes                   |
| Best For       | Repeatable tasks | Investigative tasks   |
| Complexity     | Low              | High                  |
| Cost           | Lower            | Higher                |
| Governance     | Easier           | More complex          |

# When Should You Use Prompt Chaining?

Use Prompt Chaining when:

* Steps are known in advance.
* Business process is standardized.
* Compliance requires predictable execution.
* The workflow rarely changes.

Examples:

* OCR pipeline
* Invoice processing
* Translation
* Data cleaning
* ETL


# When Should You Use Dynamic Decomposition?

Use Dynamic Decomposition when:

* The problem is open-ended.
* The investigation path is unknown.
* The system must adapt.
* Multiple tools may be required.
* The agent must reason during execution.

Examples:

* Root cause analysis
* Cybersecurity
* Software debugging
* Research
* Financial investigations


# Hybrid Architecture

Many enterprise systems combine both patterns.

Example

```text
User Request

↓

Prompt Chain

Authenticate User

↓

Validate Request

↓

Dynamic Investigation

↓

Generate Report

↓

Prompt Chain

Approval

↓

Notification
```

Fixed steps handle routine work, while dynamic decomposition is used only where reasoning and exploration are required.


# Best Practices

### Prompt Chaining

* Keep each step focused.
* Validate outputs between steps.
* Minimize branching.
* Make workflows deterministic.

### Dynamic Decomposition

* Define clear goals.
* Allow runtime planning.
* Use iterative refinement.
* Apply stopping conditions.
* Monitor execution.
* Log planning decisions.

# Interview Questions

### Q1. What is Prompt Chaining?

**Answer:**
Prompt Chaining is an orchestration pattern where a task is executed through a predefined sequence of prompts or steps. Each step depends on the previous one, and the execution path is fixed before the workflow starts.


### Q2. What is Dynamic Decomposition?

**Answer:**
Dynamic Decomposition is an orchestration pattern in which Claude analyzes the user's goal, breaks it into subtasks during execution, delegates work dynamically, evaluates progress, and adapts the plan as new information becomes available.


### Q3. When should Prompt Chaining be used?

**Answer:**
Use Prompt Chaining for predictable, repeatable business processes such as document processing, translation, invoice handling, and ETL pipelines, where the execution sequence is known in advance.


### Q4. When should Dynamic Decomposition be used?

**Answer:**
Use Dynamic Decomposition for open-ended problems where the execution path is unknown, such as incident response, research, debugging, security investigations, and root cause analysis.


### Q5. Can both patterns be combined?

**Answer:**
Yes. Many enterprise systems use a hybrid approach. Fixed, deterministic stages (authentication, validation, reporting) are implemented as prompt chains, while complex investigative stages use dynamic decomposition.

# Claude Architect Key Takeaway

For the Claude Architect exam, the distinction is:

* **Prompt Chaining** is best for **deterministic, repeatable workflows** where the sequence of operations is known before execution. It emphasizes simplicity, predictability, and governance.
* **Dynamic Decomposition** is best for **open-ended, investigative, or exploratory tasks** where Claude must determine the execution plan at runtime, adapt to new evidence, and iteratively refine its approach.

A skilled Claude architect chooses the orchestration pattern based on **task predictability**, **need for adaptation**, and **operational constraints**, rather than using the same approach for every problem.
